# 题解：忠犬PARE公 (容斥 + DP 解法)

## 1. 题目大意

给定两个长度分别为 $n$ 和 $m$ 的正整数数组 $a$ 和 $b$。要求计算以下表达式的值，并对 $10^9 + 7$ 取模：

$$ \sum_{i=1}^n \sum_{j=1}^m \frac{\text{lcm}(a_i, b_j)}{\text{gcd}(a_i, b_j)} $$

**数据范围**：
- $n, m \le 10^6$
- $\sum n \le 5 \times 10^6, \sum m \le 5 \times 10^6$
- $a_i, b_j \le 10^6$

---

## 2. 第一步：数学化简

典中典的：
$$ \text{lcm}(x, y) \times \text{gcd}(x, y) = x \times y $$
因此，我们可以将最小公倍数替换掉：
$$ \text{lcm}(x, y) = \frac{x \times y}{\text{gcd}(x, y)} $$

代入原式，题目的要求就变成了计算：
$$ \sum_{i=1}^n \sum_{j=1}^m \frac{a_i \times b_j}{\text{gcd}(a_i, b_j)^2} $$

由于这里是双重循环遍历数组，而数组元素的值域并不是很大（$\le 10^6$），在算法竞赛中，这种“值域有限制”的问题通常提示我们可以**转为通过“权值”来处理**。

我们定义：
- $va[x]$ 表示在数组 $a$ 中，所有等于 $x$ 的元素之**和**。
- $vb[y]$ 表示在数组 $b$ 中，所有等于 $y$ 的元素之**和**。

那么原式可以转化为在值域上进行枚举：
$$ \sum_{x} \sum_{y} \frac{va[x] \times vb[y]}{\text{gcd}(x, y)^2} $$

---

## 3. 第二步：定义状态，从“约数”切入

原式中最难处理的是分母上的 $\text{gcd}(x, y)$。直接求 $\text{gcd}$ 是很慢的，所以我们换一个思路考虑：**能不能枚举公约数？**

我们定义两个状态：
- **$G(d)$**：表示满足 $x$ 是 $d$ 的倍数，且 $y$ 也是 $d$ 的倍数时，所有的 $va[x] \times vb[y]$ 之和。
- **$F(d)$**：表示满足条件  $\text{gcd}(x, y)$ **恰好等于** $d$ 的所有 $va[x] \times vb[y]$ 之和。

最终我们要求的答案，就是把每一种“恰好的最大公约数”对应的总和算出来，然后除以它的平方：
$$ \text{Ans} = \sum_{d=1}^{max} \frac{F(d)}{d^2} $$

---

## 4. 第三步：如何快速求 $G(d)$ 与 $F(d)$

### 求 $G(d)$：
$G(d)$ 的条件非常宽松，只要求是 $d$ 的倍数即可，根本不在乎最大公约数到底是不是 $d$：
$$ G(d) = \left( \sum_{d|x} va[x] \right) \times \left( \sum_{d|y} vb[y] \right) $$
计算这个非常简单：对于任意的 $d$，我们只需要把它的倍数 $d, 2d, 3d \dots$ 在 $va$ 和 $vb$ 中对应的值分别加起来，然后两者相乘即可。类似质数筛（埃式筛）。

### 求 $F(d)$（容斥）：
$F(d)$ 要求最大公约数**恰好**是 $d$。那它跟 $G(d)$ 有什么关系呢？
$G(d)$ 里面其实包含了所有公约数为 $d, 2d, 3d, 4d \dots$ 的情况。因此有：
$$ G(d) = F(d) + F(2d) + F(3d) + \dots = \sum_{k=1} F(k \times d) $$

那么，我们要怎么求 $F(d)$ 呢？
直接移项：
$$ F(d) = G(d) - \sum_{k=2} F(k \times d) $$

**倒着DP**：
既然 $F(d)$ 依赖于后面更大的 $F(k \times d)$，我们完全可以**从大到小（从 $max$ 倒着遍历到 1）计算 $F(d)$**！
当我们准备求 $F(d)$ 时，所有比 $d$ 大的 $F$ 值一定都已经被正确计算并且保存下来了。我们只需要先算出 $G(d)$，然后减去所有 $d$ 的倍数的 $F$ 值，就能得到准确的 $F(d)$。

至此**用容斥 + DP 就避免了莫比乌斯反演**。

---

## 5. 复杂度

**算法流程**：
1. 预处理：用快速幂预处理出 $1 \dots 10^6$ 的逆元以及**平方逆元** `inv2`（因为分母上是 $d^2$）。
2. 输入数据，统计 $va[x]$ 和 $vb[x]$。同时记录输入数据中的最大值 $mx$。
3. 从 $i = mx$ 递减到 $1$ 枚举 $d$：
   - 提取出 $A$ 中 $i$ 的倍数的总和 `sa`，和 $B$ 中 $i$ 的倍数的总和 `sb`。
   - $F[i] = sa \times sb \pmod{10^9+7}$ （这一步即求 $G(i)$ 并作为 $F[i]$ 的初始值）
   - 遍历 $i$ 的倍数 $i \times j, (j \ge 2)$，$F[i] = F[i] - F[i \times j]$ (容斥，减去公约数更大的情况)。
   - 把当前的 $F[i]$ 除以 $i^2$ (即乘以 `inv2[i]`)，累加入最终答案。
4. 清空。

**时间复杂度**：
对于内外两层循环（不论是算 $sa, sb$ 还是容斥减掉后面的 $F$），都类似于以下级数：
$$ \frac{N}{1} + \frac{N}{2} + \frac{N}{3} + \dots + \frac{N}{N} = O(N \ln N) $$
预处理部分为 $O(N)$。

---

## 6. 核心代码

```cpp
void solve() {
    cin >> n >> m;
    int mx = 0;
    for(int i = 0, x; i < n; i++) {
        cin >> x;
        va[x] = (va[x] + x) % P;
        if(x > mx) mx = x;
    }
    for(int i = 0, x; i < m; i++) {
        cin >> x;
        vb[x] = (vb[x] + x) % P;
        if(x > mx) mx = x;
    }
    
    i64 ans = 0;

    // 倒推求确切的最大公约数之和 F(i)
    for(int i = mx; i >= 1; i--) {
        i64 sa = 0, sb = 0;
        
        // 计算当前 i 的所有倍数权值之和 (求 G(i))
        for(int j = 1; i * j <= mx; j++) {
            sa = (sa + va[i * j]) % P;
            sb = (sb + vb[i * j]) % P;
        }
        F[i] = sa * sb % P;
    
        // 容斥：减去所有以 i 的倍数作为“恰好gcd”的 F 值
        for(int j = 2; i * j <= mx; j++) {
            F[i] = (F[i] - F[i * j] + P) % P;
        }
        
        // 累加入最终答案：F[i] / i^2
        ans = (ans + F[i] * inv2[i]) % P;
    }
    cout << ans << '\n';
    
    // 清空状态(注意只清空用过的部分，避免memset超时)
    for(int i = 0; i <= mx; i++) va[i] = vb[i] = F[i] = 0;
}
```
